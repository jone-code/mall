# Mall 运营体验打磨计划

> 基于当前 `admin-web` 与管理端后端实现情况整理。  
> 目标：让管理后台从「功能齐全」升级为「日常运营顺手」。

---

## 一、总体目标

- 查单、发货、备货不卡顿
- 主控台能驱动待办，而不只是看数字
- 登录续期、权限、审计等基础体验闭环
- 会员、商品、评价等运营场景补全

**贯穿原则**

1. 先写简要设计 → 后端领域服务 → admin-service 代理（如需）→ admin-web
2. 每个 Sprint 结束要有可演示的端到端改进
3. 保持 YAGNI，不做营销、多商家等超出范围的能力

---

## 二、当前基线（已完成）

| 模块 | 已有能力 |
|------|----------|
| **主控台** | 今日订单/GMV、订单状态分布、商品概况 |
| **订单** | 筛选、详情、发货、关单、退款、运营备注、前端 CSV 导出 |
| **商品** | 批量上下架、SKU 编辑、库存调整 |
| **权限** | 角色/权限 CRUD、管理员创建与角色分配 |
| **审计** | 多条件筛选 + 详情弹窗，关键写操作有 `@AuditAction` |
| **Token** | `http.ts` 401 自动 refresh |
| **卡密/核销** | 导入、统计 tag、服务核销页 |
| **评价** | 列表 + 隐藏 |

## 三、待优化项概览

### P1 高频痛点（建议优先）

| 项 | 现状问题 | 优化方向 |
|----|----------|----------|
| 列表分页 | 订单/商品/会员等多数无分页，一次拉全量 | 统一 `page/size/total` 协议 |
| 主控台 | 只有静态数字，无待办与预警 | 待办卡片 + 点击跳转筛选列表 |
| 7 天免登录 | Token 存 `sessionStorage`，关标签即失效 | `rememberMe` 用 `localStorage` + 静默 refresh |
| 会员运营 | 仅列表 + 改状态，无详情 | 详情抽屉：地址、近期订单 |

### P2 日常运营效率

| 项 | 优化方向 |
|----|----------|
| 订单 | 订单号/手机号搜索、操作时间线、服务端条件导出、批量发货 |
| 商品 | 图片上传（替代手填 URL）、商品复制、库存变更记录、导出 |
| 卡密/核销 | 按商品库存概览、导入结果报告、脱敏展示、扫码核销 |
| 评价 | 多维筛选、恢复展示、晒图、差评预警 |

### P3 安全与合规

| 项 | 优化方向 |
|----|----------|
| 审计 | 登录/登出/refresh 落审计、CSV 导出、敏感字段脱敏、按 targetId 反查 |
| 权限 | 统一 `v-perm`、无权限友好提示、高危操作二次确认 |

### P4 体验细节（可穿插）

空态/骨架屏、订单号复制、相对时间、快捷键、页内角标通知、环境标识强化、核销页移动端适配。

## 四、分阶段计划

### Sprint 1（约 2 周）— P1 高频痛点

**目标**：解决性能瓶颈和最高频运营动作。

#### 1.1 列表分页与筛选统一

| 任务 | 范围 | 交付物 |
|------|------|--------|
| 定义统一分页协议 | 后端 common 或各服务 | `page/size/total/list` 响应结构（对齐审计、评价） |
| 订单列表分页 | `order-service` + `Orders.vue` | 分页 API；筛选：状态、用户 ID、订单号、日期范围、商品类型 |
| 商品列表分页 | `product-service` + `Products.vue` | 分页 API；筛选：关键词、状态、类目、类型 |
| 会员列表分页 | `admin-service` + `Members.vue` | 分页 + 注册时间、状态筛选 |
| 卡密/核销码分页 | `order-service` + `Cards.vue` / `ServiceCodes.vue` | 避免大数据量一次拉全量 |

**验收**

- [ ] 订单 > 500 条时列表加载 < 1s
- [ ] 所有主列表有分页组件
- [ ] 筛选变更后重置到第 1 页

#### 1.2 主控台待办驱动

| 任务 | 范围 | 交付物 |
|------|------|--------|
| 待办统计 API | `order-service` / `admin-service` | 待发货、待支付、退款中、卡密不足、核销码不足数量 |
| 待办卡片 UI | `Dashboard.vue` | 可点击跳转对应列表（query 预填筛选） |
| 库存预警 | product + 卡密/核销码 stats | 低库存 SKU、可用卡密低于阈值时 warning |

**验收**

- [ ] 主控台「待发货 N」点击后进入已筛选订单列表
- [ ] 卡密不足时有明显预警，可跳转卡密池

#### 1.3 登录续期体验闭环

| 任务 | 范围 | 交付物 |
|------|------|--------|
| Token 存储策略 | `admin.ts` + `http.ts` | `rememberMe` → `localStorage`；否则 `sessionStorage` |
| 启动静默 refresh | `router` 或入口 | 有 refreshToken 时自动续期 |
| 活跃会话管理（可选） | `admin-service` + 新页面 | 查看/踢出其他设备（对齐 C 端 `/me/sessions`） |

**验收**

- [ ] 勾选「7 天免登录」后关浏览器重开仍可访问
- [ ] accessToken 过期无感 refresh，仅 refresh 失效才跳登录

#### 1.4 会员详情页

| 任务 | 范围 | 交付物 |
|------|------|--------|
| 会员详情 API | `admin-service` | 基本信息 + 地址列表 + 近期订单 |
| 详情抽屉/页 | `Members.vue` | 点击会员行打开详情 |
| 状态变更增强 | 前端 + 审计 | 禁用/注销二次确认 + 原因（可选） |

**验收**

- [ ] 客服可查看会员最近 10 笔订单
- [ ] 可查看收货地址列表

---

### Sprint 2（约 2 周）— P2 日常运营效率

**目标**：提升商品、订单、卡密、评价的日常操作效率。

#### 2.1 订单运营深化

| 任务 | 优先级 | 交付物 |
|------|--------|--------|
| 订单号/手机号搜索 | 高 | 列表快速查单 |
| 订单操作时间线 | 高 | 详情展示 `order_log` 状态变更 |
| 服务端条件导出 | 高 | 按筛选导出 CSV/Excel |
| 批量发货 | 中 | 勾选多个 `PAID` 实物单批量填物流 |
| 物流轨迹对接 | 低 | 替换 Mock，接快递查询或手动维护 |
| 列表「拒绝退款」 | 低 | 与详情页能力对齐 |

#### 2.2 商品运营深化

| 任务 | 优先级 | 交付物 |
|------|--------|--------|
| 图片上传 | 高 | MinIO 或本地存储 + 拖拽排序 |
| 商品复制 | 中 | 基于已有 SPU 快速创建 |
| 库存变更记录 | 中 | 调整人、时间、数量（可先用审计） |
| 商品导出 | 低 | SKU 维度 Excel |

#### 2.3 卡密 / 核销码运营

| 任务 | 优先级 | 交付物 |
|------|--------|--------|
| 按商品维度库存概览 | 高 | 可用/已发放/缺口一览 |
| 导入结果报告 | 高 | 成功/重复/失败 + 原因 |
| 卡密脱敏 + 复制 | 中 | 列表默认遮罩，授权后可查看 |
| 扫码核销 | 低 | 服务核销页支持扫码枪/摄像头 |

#### 2.4 评价管理增强

| 任务 | 优先级 | 交付物 |
|------|--------|--------|
| 多维筛选 | 高 | 商品、评分、时间、状态 |
| 恢复展示 | 中 | `unhide` 接口 + 按钮 |
| 晒图查看 | 低 | 接 MinIO 后展示评价图 |
| 差评预警 | 低 | 1–2 星评价主控台提醒 |

#### 2.5 主控台趋势图（可与 2.1 并行）

| 任务 | 交付物 |
|------|--------|
| 趋势统计 API | 近 7/30 天订单量、GMV 按日聚合 |
| 趋势图 UI | `Dashboard.vue` 折线图 |
| 统计卡片穿透 | 今日订单 → 今日列表；今日新用户 → 会员列表 |

---

### Sprint 3（约 1–2 周）— P3 安全与合规

#### 3.1 审计补全

| 任务 | 交付物 |
|------|--------|
| 登录/登出/refresh/密码重置落审计 | 后端审计写入 |
| 审计 CSV 导出 | `Audit.vue` 导出按钮 |
| 敏感字段脱敏 | 请求体密码、卡密打码 |
| 按 targetId 反查 | 支持订单号等反查操作记录 |

#### 3.2 权限与操作体验

| 任务 | 交付物 |
|------|--------|
| 统一 `v-perm` 指令 | 按钮级权限 |
| 无权限友好提示 | 「需要 xxx 权限」 |
| 高危操作二次确认 | 批量下架、同意退款、批量导入等 |
| 菜单角标（可选） | 侧边栏「订单管理」显示待办数 |

---

### Sprint 4（按需）— P4 体验细节

可与前面 Sprint 穿插进行。

| 类别 | 任务 |
|------|------|
| 空态/骨架屏 | 各列表统一 loading、empty 组件 |
| 表格体验 | 订单号复制、金额右对齐、相对时间 |
| 快捷键 | Enter 搜索、Esc 关弹窗 |
| 消息通知 | 页内角标或浏览器 Notification |
| 环境标识 | `EnvBadge` 区分 dev/staging/prod |
| 移动端 | 服务核销页平板/手机可用 |

## 五、推荐执行顺序

```
Sprint 1
  1. 订单分页 API + Orders.vue
  2. 商品分页 API + Products.vue
  3. Dashboard 待办卡片 + 跳转
  4. rememberMe → localStorage
  5. 会员详情抽屉

Sprint 2
  6. 订单：搜索 + 时间线 + 服务端导出
  7. 商品：图片上传
  8. 卡密：导入报告 + 商品维度概览
  9. 评价：筛选 + 恢复展示
  10. Dashboard 趋势图

Sprint 3
  11. 审计补全 + 导出
  12. 权限指令 + 二次确认

Sprint 4
  13. 体验细节（按反馈择优）
```

## 六、标准工作流

每个任务按项目既有 Phase 流程执行：

```
1. 补设计说明（API 清单 + 交互要点，小改可 inline 注释）
2. 更新 schema.sql / patch-*.sql（如有 DDL）
3. 实现领域服务 + 单测
4. admin-service 代理 + 权限码（如需）
5. admin-web 页面 + api 模块
6. deploy README 联调步骤（如有新依赖）
7. 勾选验收 checklist
```

## 八、刻意不做（YAGNI）

- 优惠券 / 秒杀 / 拼团
- 多商家入驻
- 完整 BI 报表系统
- 实时 WebSocket 推送（首期轮询或手动刷新即可）
- 物流自营
